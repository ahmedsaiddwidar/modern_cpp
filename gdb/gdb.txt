ğŸŸ¥ GDB

GDB is neither a build system nor a build tool.

GDB is a debugger.

It lets you examine your program while it is running or crashed.

You use it to:

Run your program step by step

Set breakpoints

See variable values

Inspect memory/registers

Debug segmentation faults

Backtrace crashes

Watch variable changes

Debug multi-threaded programs

Debug optimized code

Example commands:

gdb ./app
(gdb) break main
(gdb) run
(gdb) next
(gdb) print x
(gdb) backtrace
